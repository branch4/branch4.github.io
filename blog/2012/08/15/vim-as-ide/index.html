Vim as an IDE

Aug 15th, 2012 | Comments


I started building VIM environment.

And I’m already forgetting it, so I’d have to leave a note for myself in the future…

OK, I installed these plugins.
vundle
project
unite
git-vim
quickrun
vim-ref
taglist
1. install Vundle

$ git clone http://github.com/gmarik/vundle.git ~/.vim/bundle/vundle.git
edit .vimrc.

add the following lines.


“” Vundle config
set nocompatible
filetype off
set rtp+=~/.vim/bundle/vundle/
call vundle#rc()
Bundle ‘gmarik/vundle’

Bundle ‘Shougo/unite.vim’
Bundle ‘thinca/vim-ref’
Bundle ‘thinca/vim-quickrun’
Bundle ‘Shougo/git-vim’
Bundle ‘project.tar.gz’
filetype plugin indent on

2. boot vim and exec :BundleInstall

wait for the mini buffer window to say “Done”.

3. configure vim-ref(download php manual/install w3m or lynx)

Download  PHP manual.
You’ll need the “Many files” version.

Just extracted in some place.

$ sudo apt-get install w3m

4. edit .vimrc

add the following lines.


“” vim-ref config
let g:ref_phpmanual_path = $HOME . ‘/work/PHP_manual/php-chunked-xhtml/’
let $PATH = $PATH . ‘:/usr/bin/’
let g:ref_phpmanual_cmd  = ‘w3m -dump %s’

hit on the function you want to check.
Vim will automatically split the window and show the reference.

5. using project

open vi.
exec :Project
exec \c
answer the questions
save with :w
when you want to comeback, open vi and exec :Project

6. using quickrun

write your code
exec \r

I did start using some other function as omni completion,
but I will write about them next time.
I’m so sleepy…
