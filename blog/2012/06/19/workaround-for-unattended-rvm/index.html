Workaround for Unattended Rvm Installation of Ruby

Jun 19th, 2012 | Comments

I found a workaround.

The installation script is checking for .rvm/rubies directory for an another ruby installation.

If there is another ruby directory there, it wont display that noisy requirements.

So I mkdired a directory before the installation.

mkdir ${HOME}/.rvm/rubies/ruby-1.9.3-p125
rvm reinstall 1.9.3-p125

It has to be, rvm *re*install 1.9.3-p125 , if you mkdired a same version.

there are two easier ways to automate installing rubies, first:

$rvm_path/bin/rvm install 1.9.3

second:

source $rvm_path/scripts/rvm
command rvm install 1.9.3

they will not set default ruby so you need to set it manually:

rvm alias create default 1.9.3
