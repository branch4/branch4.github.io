[XML] Unicode バイト順マークがないため、Unicode に切り替えられません。

Oct 28th, 2013 | Comments

XMLEDITOR.NETで、xmlファイルを開いたら、こんなエラーが。

「Unicode バイト順マークがないため、Unicode に切り替えられません。」

ファイル開いてみたら、なぜかutf-16になってたので、utf-8に修正しました。

修正前 
  <?xml version="1.0" encoding="utf-16"?>
修正後 
  <?xml version="1.0" encoding="utf-8"?>
あとはちゃんと開けました。ちゃんちゃん。

[XML] There Is No Unicode Byte Order Mark. Cannot Switch to Unicode.

Oct 28th, 2013 | Comments

Tried to open a xml file with XMLEDITOR.NET.

And the following error occured. 
  There is no Unicode byte order mark. Cannot switch to Unicode.
I checked the xml file, and fixed the line which said, 
  <?xml version="1.0" encoding="utf-16"?>
to 
  <?xml version="1.0" encoding="utf-8"?>
wrong : utf-16
correct: utf-8

And now I could open it.

[MySQL] 文字化け修正

Oct 22nd, 2013 | Comments

mysqlのバージョンは、5.5.32-0ubuntu0.12.04.1。

インストールしたてのmysqlに、何も考えず

$ rake db:seed

やったら、がっつり「？？」に文字化けしちゃった。
毎度設定忘れるので、メモ。

mysql> select  * from states;
+—-+——-+———————+———————+
| id | state | created_at          | updated_at          |
+—-+——-+———————+———————+

|  2 | ??    | 2013-10-22 07:06:42 | 2013-10-22 07:06:42 |
|  3 | ??    | 2013-10-22 07:06:42 | 2013-10-22 07:06:42 |
|  4 | ??    | 2013-10-22 07:06:42 | 2013-10-22 07:06:42 |
|  5 | ??    | 2013-10-22 07:06:42 | 2013-10-22 07:06:42 |
|  6 | ??    | 2013-10-22 07:06:42 | 2013-10-22 07:06:42 |
|  7 | ??    | 2013-10-22 07:06:42 | 2013-10-22 07:06:42 |
+—-+——-+———————+———————+
7 rows in set (0.01 sec)


my.cnfを編集して追記する。
$ sudo vi /etc/mysql/my.cnf

[mysqld]の欄に下記追記。
character-set-server = utf8
skip-character-set-client-handshake
collation-server = utf8_general_ci
init-connect = SET NAMES utf8

そしてmysql再起動。
$ sudo /etc/init.d/mysql restart

database作り直し。
$ mysql
mysql> create database hogehoge

$ rake db:migrate
$ rake db:seed

mysql> select * from states;
