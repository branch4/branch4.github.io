[RoR] Excluding Active Record or Some Other Modules

Apr 4th, 2012 | Comments


I wanted to exclude Active Record in my test app of rails.

# I would probably use DB connections in almost every apps I develop,
# but I didn’t want to setup databases for my test app.

In order to do this, Mr.Google told me to change below files.


- config/environments/development.rb
- config/environments/production.rb


to edit config.frameworks and delete active_record, like below example.

- before
  config.frameworks -= [ :active_record, :active_resource, :action_mailer ]
- after

  config.frameworks -= [ :active_resource, :action_mailer ]


which didn’t exist in my development.rb/production.rb files.

Next googling took me to the answer.

probably, in the former version, above solution was correct.

But in the recent version(I don’t know from which version)  the solution is below.

(refer to http://therailworld.com/posts/27-Disable-ActiveRecord-in-Rails-3)

File to edit : config/application.rb.

- before
  require “rails/all”

- after
  # require “rails/all”  <- comment out

  # add the lines below and comment out what you don’t need
  # Pick the frameworks you want:
  # require “active_record/railtie”
  require “action_controller/railtie”
  require “action_mailer/railtie”
  require “active_resource/railtie”
  require “rails/test_unit/railtie”

keep in mind that you must exclude configs related to the modules commented out.
So, when I comment active_records out, I also commented out some configs in
config/environment/development.rb and config/application.rb.

Ruby on Rails - rvm(Ruby Version Manager) -

Mar 19th, 2012 | Comments

This entry is a memo about developing with ruby on rails.

Prerequisites:
  - I’ve never used ruby
  - I’ve developed with MVC Framework in Java and Python

First, I did some googling about ruby and etc.

And decided to learn about following applications.

  - rvm(Ruby Version Manager)
  - rubygems
  - rails
  - passenger

From above, this entry focuses on RVM.

RVM(Ruby Version Manager) manages ruby version and gemsets.

You can install many versions of rubys, and many combinations of gemsets,

and choose to use a right version on right purposes.

1. installing rvm
go home and install it.


$ cd
$ bash < <(curl -s https://rvm.beginrescueend.com/install/rvm)

You’ll get
Deprecated command, please use:

bash -s stable < <(curl -s https://raw.github.com/wayneeseguin/rvm/master/binscripts/rvm-installer)

So, execute the command above, instead.

This installs rvm under ${HOME}/.rvm

2. setup environment variables

Edit .bashrc and add this line.

  [[ -s “$HOME/.rvm/scripts/rvm” ]] && source “$HOME/.rvm/scripts/rvm”

3. check ruby version that can be installed and install some

“rvm list known” command shows what’s available.

$ rvm list known
# MRI Rubies

[ruby-]1.8.6[-p420]
[ruby-]1.8.7[-p358]
[ruby-]1.8.7-head
[ruby-]1.9.1[-p431]
[ruby-]1.9.2-p180
[ruby-]1.9.2-p290
[ruby-]1.9.2[-p318]
[ruby-]1.9.2-head
[ruby-]1.9.3-preview1
[ruby-]1.9.3-rc1
[ruby-]1.9.3-p0
[ruby-]1.9.3[-p125]
[ruby-]1.9.3-head
ruby-head
…snip

I installed 1.9.3-p125 and 1.8.7-p358.

$ rvm install 1.9.3-p125
$ rvm install 1.8.7-p358

4. set default ruby

$ rvm use 1.9.3-p125 –default
$ ruby -v <- check the version

ruby 1.9.3p125 (2012-02-16 revision 34643) [x86_64-linux]


5. create gemsets

gemset is a grouping function of gems.

I can create gemset for rails version3.1 and rails version 3.2,

and use it on different purpose, when ever I need to.

$ rvm gemset create rails31

$ rvm gemset create rails32


$ rvm gemset list
gemsets for ruby-1.9.3-p125 (found in ${HOME}/.rvm/gems/ruby-1.9.3-p125)
   global
   rails31
   rails32



6. set default gemsets
$ rvm use 1.9.3-p125@rails32 –default
7. install passenger and rails3.2.2
check where you are.
$ rvm gemset list
gemsets for ruby-1.9.3-p125 (found in ${HOME}/.rvm/gems/ruby-1.9.3-p125)
   global
   rails31
=> rails32
$ gem install rails –version 3.2.2
$ gem install passenger
8. setup apache to use passenger(install modules and so on)
a. if you use apt-get installed apache
just type
$ passenger-install-apache2
and follow the instructions.
b. if you plan to use self compiled apache
  - add apache binary path to PATH
    PATH=${PATH}:/usr/local/httpd/bin/
    export PATH
  - set APXS2 path
  - set APR_CONFIG path
  - set APU_CONFIG path
    APXS2=/usr/local/httpd/bin/apxs
    APR_CONFIG=/usr/local/httpd/bin/apr-1-config
    APU_CONFIG=/usr/local/httpd/bin/apu-1-config
    export APXS2
    export APR_CONFIG
    export APU_CONFIG
  - set HTTPD path
    HTTPD=/usr/local/httpd
    export HTTPD
and then type
$ rvmsudo passenger-install-apache2
and do what the terminal saids.
