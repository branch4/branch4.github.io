Etckeeper Howto

Jun 8th, 2012 | Comments

I used etckeeper to backup and version control my files under /etc.

# in precise pangolin

install
$ sudo apt-get install etckeeper

edit config file to use git(default uses bzr)
$ sudo vi /etc/etckeeper/etckeeper.conf

before edit

# The VCS to use.
#VCS=”hg”
#VCS=”git”   <--- uncomment this line
VCS=”bzr”     <--- comment out this line
#VCS=”darcs”

after edit

# The VCS to use. #VCS=”hg”
VCS=”git”   <--- uncomment this line
#VCS=”bzr”     <--- comment out this line
#VCS=”darcs”
initialize repository
$ sudo etckeeper init
this creates and initializes git repository in /etc.
it also adds the files in /etc to the staging, so if you want to
exclude any files to be managed, you should follow the following
instructions.
remove files in staging
$ cd /etc
$ sudo git rm –cached -r -f ./
I’m going to use gitignore, and gitignore ignores a file only if its not in the staging.
thats why I removed them.
edit .gitignore(its already gonna be there, generated by etckeeper)
$ cd /etc
$ sudo vi .gitignore
what I will do in the ignore file is
- ignore all files
- explicitly point out files I want to manage

following .gitignore only commits these files/directories.

libvirt
compizconfig
bash_completion
environment

when I installed etckeeper, it installed cron task to take a backup,
once every day, so it will be backedup everyday.
$ cat /etc/cron.daily/etckeeper 
#!/bin/sh
set -e
if [ -x /usr/bin/etckeeper ] && [ -e /etc/etckeeper/etckeeper.conf ]; then
. /etc/etckeeper/etckeeper.conf
if [ “$AVOID_DAILY_AUTOCOMMITS” != “1” ]; then
# avoid autocommit if an install run is in progress
lockfile=/var/cache/etckeeper/packagelist.pre-install
if [ -e “$pe” ] && [ -n “$(find “$lockfile” -mtime +1)” ]; then
rm -f “$lockfile” # stale
fi
if [ ! -e “$lockfile” ]; then
AVOID_SPECIAL_FILE_WARNING=1
export AVOID_SPECIAL_FILE_WARNING
if etckeeper unclean; then
etckeeper commit “daily autocommit” >/dev/null
fi
fi
fi
fi
I also wanted to push to github after commit, so I set it up.
create a script
$ sudo vi /etc/etckeeper/commit.d/40git-push
=========beginngin of 40git-push=========
#!/bin/sh
set -e
if [ “$VCS” = git ] && [ -d .git ]; then
  cd /etc/
  git push origin master
fi
========= end of 40git-push=========
test
$ sudo vi /etc/environment
PATH=”/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games”
EDITOR=”/usr/bin/vim”   <-- add this line
commit
$ sudo etckeeper commit “test”
if it gets commited to your local repository and github, its success.

[RoR] Excluding Active Record or Some Other Modules

Apr 4th, 2012 | Comments


I wanted to exclude Active Record in my test app of rails.

# I would probably use DB connections in almost every apps I develop,
# but I didn’t want to setup databases for my test app.

In order to do this, Mr.Google told me to change below files.


- config/environments/development.rb
- config/environments/production.rb


to edit config.frameworks and delete active_record, like below example.

- before
  config.frameworks -= [ :active_record, :active_resource, :action_mailer ]
- after

  config.frameworks -= [ :active_resource, :action_mailer ]


which didn’t exist in my development.rb/production.rb files.

Next googling took me to the answer.

probably, in the former version, above solution was correct.

But in the recent version(I don’t know from which version)  the solution is below.

(refer to http://therailworld.com/posts/27-Disable-ActiveRecord-in-Rails-3)

File to edit : config/application.rb.

- before
  require “rails/all”

- after
  # require “rails/all”  <- comment out

  # add the lines below and comment out what you don’t need
  # Pick the frameworks you want:
  # require “active_record/railtie”
  require “action_controller/railtie”
  require “action_mailer/railtie”
  require “active_resource/railtie”
  require “rails/test_unit/railtie”

keep in mind that you must exclude configs related to the modules commented out.
So, when I comment active_records out, I also commented out some configs in
config/environment/development.rb and config/application.rb.
