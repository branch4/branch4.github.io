Ruby on Rails - rvm(Ruby Version Manager) -

Mar 19th, 2012 | Comments

This entry is a memo about developing with ruby on rails.

Prerequisites:
  - I’ve never used ruby
  - I’ve developed with MVC Framework in Java and Python

First, I did some googling about ruby and etc.

And decided to learn about following applications.

  - rvm(Ruby Version Manager)
  - rubygems
  - rails
  - passenger

From above, this entry focuses on RVM.

RVM(Ruby Version Manager) manages ruby version and gemsets.

You can install many versions of rubys, and many combinations of gemsets,

and choose to use a right version on right purposes.

1. installing rvm
go home and install it.


$ cd
$ bash < <(curl -s https://rvm.beginrescueend.com/install/rvm)

You’ll get
Deprecated command, please use:

bash -s stable < <(curl -s https://raw.github.com/wayneeseguin/rvm/master/binscripts/rvm-installer)

So, execute the command above, instead.

This installs rvm under ${HOME}/.rvm

2. setup environment variables

Edit .bashrc and add this line.

  [[ -s “$HOME/.rvm/scripts/rvm” ]] && source “$HOME/.rvm/scripts/rvm”

3. check ruby version that can be installed and install some

“rvm list known” command shows what’s available.

$ rvm list known
# MRI Rubies

[ruby-]1.8.6[-p420]
[ruby-]1.8.7[-p358]
[ruby-]1.8.7-head
[ruby-]1.9.1[-p431]
[ruby-]1.9.2-p180
[ruby-]1.9.2-p290
[ruby-]1.9.2[-p318]
[ruby-]1.9.2-head
[ruby-]1.9.3-preview1
[ruby-]1.9.3-rc1
[ruby-]1.9.3-p0
[ruby-]1.9.3[-p125]
[ruby-]1.9.3-head
ruby-head
…snip

I installed 1.9.3-p125 and 1.8.7-p358.

$ rvm install 1.9.3-p125
$ rvm install 1.8.7-p358

4. set default ruby

$ rvm use 1.9.3-p125 –default
$ ruby -v <- check the version

ruby 1.9.3p125 (2012-02-16 revision 34643) [x86_64-linux]


5. create gemsets

gemset is a grouping function of gems.

I can create gemset for rails version3.1 and rails version 3.2,

and use it on different purpose, when ever I need to.

$ rvm gemset create rails31

$ rvm gemset create rails32


$ rvm gemset list
gemsets for ruby-1.9.3-p125 (found in ${HOME}/.rvm/gems/ruby-1.9.3-p125)
   global
   rails31
   rails32



6. set default gemsets
$ rvm use 1.9.3-p125@rails32 –default
7. install passenger and rails3.2.2
check where you are.
$ rvm gemset list
gemsets for ruby-1.9.3-p125 (found in ${HOME}/.rvm/gems/ruby-1.9.3-p125)
   global
   rails31
=> rails32
$ gem install rails –version 3.2.2
$ gem install passenger
8. setup apache to use passenger(install modules and so on)
a. if you use apt-get installed apache
just type
$ passenger-install-apache2
and follow the instructions.
b. if you plan to use self compiled apache
  - add apache binary path to PATH
    PATH=${PATH}:/usr/local/httpd/bin/
    export PATH
  - set APXS2 path
  - set APR_CONFIG path
  - set APU_CONFIG path
    APXS2=/usr/local/httpd/bin/apxs
    APR_CONFIG=/usr/local/httpd/bin/apr-1-config
    APU_CONFIG=/usr/local/httpd/bin/apu-1-config
    export APXS2
    export APR_CONFIG
    export APU_CONFIG
  - set HTTPD path
    HTTPD=/usr/local/httpd
    export HTTPD
and then type
$ rvmsudo passenger-install-apache2
and do what the terminal saids.
I’ll write about executing self made ruby app on rails on the next entry.

Managing KVM Image With Virt-manager

Mar 12th, 2012 | Comments

# Some information added on 2012/4/1(at the end of an article)

On my last post, I wrote about installation of Slackware on KVM.

I used kvm command.

Today, I’ll write about managing KVM images.

It looks like virsh(Command line) or virt-manager(GUI) is popular.

First, about virt-manager.

1. install virt-manager
# apt-get install virt-manager

2. boot virt-manager
I had a minor trouble where virt-manager complained about not being
able to connect to the local QEMU.

It told me to
- check if I had installed libvirtd
- check if the user was in the libvirtd group
- check if the libvirtd was started

I checked, and the user wasn’t in the libvirtd group, which I added with
# adduser libvirtd

rebooted the whole system
# reboot

and fixed the matter.

3. set the correct storage directory to virt-manager
On default, virt-manager is set to look for

/var/lib/libvirt/images

for the virtualized images.

On my last post, I used virt-install  kvm command, and placed the image to

/usr/local/kvs_images

so I had to change that first.

To do that, edit,

/etc/libvirt/storage/default.xml

change the following to

before
/var/lib/libvirt/images

after

/usr/local/kvs_images

and reboot libvirtd.
# service libvirt-bin restart
and you’ll be able to see the images you created with virt-install.

4. setup xml file for slackware guest
create a test file with kvm command in it.
In my case this is it.

———————test.txt———————
kvm -drive file=/usr/local/kvm_images/slak01.img,if=virtio,boot=on -boot d -m 512 -monitor stdio

————————————————–

Hit command below.
# virsh domxml-from-native qemu-argv test.txt > test.xml

Edit some lines.
Especially, line as follows.


/usr/bin/qemu-system-x86_64

Execute,
# virsh
# define slak01 # in the virsh prompt

And you’ll now be able to use your slak from virt-manager!

That’s all for today.
Byebye.


# added on 2012/4/2

When managing KVM image on virt-manager(on ubuntu),

there is a configuration file on /etc/libvirt/qemu/*.xml.

It’s called a Domain XML.

And the format is in the web site below.

http://libvirt.org/formatdomain.html

Slackware on KVM

Mar 11th, 2012 | Comments

It’s probably nothing new.

Below are the sites that I checked.
http://www.grocock.me.uk/blog/booting-slackware-from-a-virtio-hard-disk-in-a-kvm-virtual-machine/
http://www.linuxquestions.org/questions/slackware-14/slackware64-current-kvm-guest-boot-issue-871981/

1. check your CPU for HW support

$ grep -E ‘vmx|svm’ /proc/cpuinfo

2. install kvm and bridge-utils
# apt-get install kvm bridge-utils

3. Add user (that executes kvm) to group kvm
# adduser kvm

4. prepare images
$ kvm-img create -f qcow .img 10G

5. prepare guest OS images or CDs

I used ISO file.
# mount -t iso9660 -o loop

6. execute kvm
$ kvm -drive file=/usr/local/kvm_images/slak01.img,if=virtio,boot=on -cdrom /usr/local/os_images/slackware64-13.37-install-dvd.iso -boot d -m 512 -monitor stdio

Then, you’ll be seeing Slackware installer boot.
Now, never forget to use virtio.
You need this to install Slackware.

7. install Slackware
You do this, Just as you install Slack on physical server,
except that you’ll fail to install LILO.
* never choose btrfs as a filesystem for /boot, lilo don’t like it

8. exit setup(we’re gonna install LILO from here)

9. mount devices/sys/proc -> chroot
# mount -obind /dev /mnt/dev

# mount -obind /sys /mnt/sys
# mount -t proc proc /proc /mnt/proc chroot /mnt

10. make initrd

Edit mkinitrd.conf
# cd /etc/
# cp mkinitrd.conf.sample mkinitrd.conf
# vi mkinitrd.conf

Edit following lines:
MODULE_LIST=”ext4:virtio_blk:virtio_pci”
ROOTFS=”ext4”
ROOTDEV=”/dev/vda3”

Get a command line for making initrd.
# /usr/share/mkinitrd/mkinitrd_command_generator.sh -r
#

11. edit lilo.conf
# vi /etc/lilo.conf

Add following line after boot = /dev/vda
disk = /dev/vda bios=0x80 max-partitions=7

Add following line after image = /boot/vmlinuz
initrd = /boot/initrd.gz

Comment out root = /dev/vda3.

12. install LILO
# lilo

13. reboot
# reboot

14. pray it to boot up

That’s all for today.
That did it for me, I think.

[MySQL] Show Database Character-set and Change It to Utf8

Dec 11th, 2011 | Comments

> show variables like “char%”;

[mysqld]
#character-set-server = latin1
#collation-server = latin1_general_ci
default-character-set=utf8
skip-character-set-client-handshake

character-set-server defines default character-set.
collation-server defines servers default check order for character-set.
